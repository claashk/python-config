Context Implementation¶

Context objects form the heart of a Schema. They define how content is interpreted within certain parts of the config file.

Context API¶

class schema.Context(name='', parent=None)[source]¶

Context base class with minimal functionality

The data model consists of a tree, where each node is represented by an instance of a Context derived class. Context objects are designed to provide minimal functionality themselves. Additional functionality can be added through mixins. This allows for simple and versatile customization with little overhead.

For more information on mixins refer to the :mod:~schema.mixins sub package.
